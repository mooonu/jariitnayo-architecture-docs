[2026.06.21 / Apple 신규 가입 cold 실측 1회]

## 0. 요약

- Apple 토큰 검증 시간 **6,157ms → 958ms** (약 84% 감소)
- Apple 신규 가입 응답시간 **8,476ms → 2,517ms** (약 70% 감소, 약 3배 빠름)



## 1. 변경 내용

`httpx` → Python 표준 라이브러리 `http.client` 로 교체. 호출부 5곳 모두 변환.

| 함수 | 람다 | 변환 패턴 |
|---|---|---|
| `_verify_apple_token` | auth | `httpx.get` → `http.client GET` |
| `_verify_kakao_token` | auth | `httpx.get` + Authorization 헤더 → `http.client GET` |
| `_fetch_apple_refresh_token` | auth | `httpx.post(data=...)` → `http.client POST + urlencode` |
| `_apple_revoke` | delete_user | `httpx.post(data=...)` → `http.client POST + urlencode` |
| `_kakao_unlink` | delete_user | `httpx.post(headers=..., data=...)` → `http.client POST + Authorization + urlencode` |

추가 정리:
- `lambda/common/python/requirements.txt`에서 `httpx`, `requests` 제거 (`requests`는 어디서도 사용 안 함)
- `get_seats/handler.py`에서 미사용 `import httpx`, `urllib.parse`, `os` 제거

→ 원인 분석: [`01-auth-응답지연-분석.md`](./01-auth-응답지연-분석.md)
→ 테스트 방법: [`02-테스트방법.md`](./02-테스트방법.md)



## 2. 개선 수치 (Apple 신규 가입 cold 기준)

| 구간 | 개선 전 (httpx) | 개선 후 (http.client) | 차이 |
|---|---|---|---|
| Init Duration | 909ms | **665ms** | **-244ms** (httpx 임포트 제거) |
| `_verify_apple_token` | **6,157ms** | **958ms** | **-5,199ms (84% 감소)** |
| DDB GSI 조회 | ~800ms | ~259ms | -541ms (변동성 범위, 변경과 무관) |
| `_fetch_apple_refresh_token` | 1,415ms | ~1,180ms | -235ms |
| DDB put + 후속 | ~26ms | ~120ms | +변동성 |
| **Duration 합계** | **8,476ms** | **2,517ms** | **-5,959ms (70% 감소)** |

- 비교 기준: 개선 전 - 분석 문서 1번 "가장 느린 케이스" 8,476ms / 개선 후 - 2026-06-21 21:38 KST 실사용 회원가입 cold 1건
