# 서울시 API 호출 람다 응답지연 개선 결과

[2026.06.07]

서울시 지하철 실시간 API를 호출하는 람다 함수들의 cold start 응답지연 문제를 해결하고, 실제 배포 후 CloudWatch Logs로 검증한 결과.

변경 내용: `httpx` → `http.client` (Python 표준 라이브러리) 교체
대상 함수: `get_nearby_arrival`, `get_train`, `pass_seat`

<br>

## 0. 요약

- Cold start HTTP 요청 시간 **2,399ms → 72ms** (97% 감소, 약 33배 빠름)
- Warm HTTP 요청 시간 **183ms → 38ms** (79% 감소)
- Cold start 전체 응답시간 **3,048ms → 680ms** (78% 감소)

<br>

## 1. 개선 수치

> 람다 초기화 시간(Init Duration)은 아래 HTTP 요청 구간 수치에 포함되지 않는다.
> 개선 전후 모두 평균 약 600 ~ 650ms 수준으로 변화 없음.

| 항목 | 개선 전 (httpx) | 개선 후 (http.client) | 개선율 |
| --- | --- | --- | --- |
| Cold Start HTTP 요청 구간 | 평균 **2,399ms** | **72ms** | **97% 감소 (33배 빠름)** |
| Warm HTTP 요청 구간 | 183ms | **38ms** | 79% 감소 (4.8배 빠름) |
| Cold Start 전체 Duration | **3,048ms** | **680ms** | **78% 감소 (4.5배 빠름)** |

<br>

## 4. 의의

- 외부 라이브러리 의존성 없이 Python 표준 라이브러리만으로 97% 성능 개선
- Cold Start 시 타임아웃 오류(3,000ms 초과)가 반복되던 문제 해소
- Warm 상태에서도 추가로 79% 개선 (httpx 클라이언트 객체 생성 비용 제거)
- Provisioned Concurrency(추가 비용) 없이 소프트웨어 변경만으로 해결
